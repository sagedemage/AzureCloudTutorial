Git Tutorial

Introduction:

Git is a free and open source distributed version
control system designed to handle everything from small to very
large projects with speed and efficiency.

Installation:

  1. Install the Git client
  2. Create a Github account

Git Configuration:

To start using Git on your computer you must first enter your
credentials to identify yourself as the author of your work

  1. Open up Git Bash terminal
  2. Enter your username
    3. git config --global user.name [your_username]
  3. Enter your email
    4.  git config --global user.email [your_email]
  4. Check if Git was properly configured
    5. git configuration --global --list

How to make a repository:

A Git repository is a collection of files of various different versions of a project.
These files are imported from the repository into the local server of the user for further updations and modifications in the content of the file.
A version control system is used to create these versions and store them in a specific place called a repository.

  1. Open the Git Bash terminal
  2. Navigate to the directory of where you want the repository
  3. Create a README.md file
    4. touch README.md
  4. Create a repository
    5. git init
  5. Move README.md into staging area
    6. git add README.md
  6. Capture a snapshot of currently staged changes
    7. git commit -m "first commit"
  7. Add a remote repository
    8. git remote add origin git@github.com:your_username/.git
  8. Push changes to remote repository
    9. git push -u origin master

Git Commands

  • Commit any files you've added with the git add commit and also commits and files you've changed since then
    • git commit -a
  • Lists all files in directory
    • git status
  • Deletes the file from your working directory and stages the deletion
    • git rm [file_name]
  • lists all the local branches in the local repository
    • git branch
  • Creates new branch
    • git branch [branch_name]
  • Switches to specified branch
    • git checkout [branch_name]
  • creates new branch but also switches to it)
    • git checkout -b [branch_name]
  • Merges the specific branch's history into the current branch
    • git merge [branch_name]
  • Used to connect your local repository to the remote server
    • git remote add [variable_name] [remote_server_link]
  • Sends the commmited changes of master branch to your remote repository
    • git push [variable_name] master
  • Fetches and merges changes on the remote server to your working directory
    • git pull [repository_name]
